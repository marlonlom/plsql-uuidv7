# Contributing to plsql-uuid-v7

Thank you for your interest in contributing to the plsql-uuid-v7 project! We welcome contributions from the community. Please follow the guidelines below to ensure a smooth contribution process.

## How to Contribute

1. **Fork the Repository**: Start by forking the repository to your own GitHub account.

2. **Clone Your Fork**: Clone your forked repository to your local machine.
   ```
   git clone https://github.com/your-username/plsql-uuid-v7.git
   ```

3. **Create a Branch**: Create a new branch for your feature or bug fix.
   ```
   git checkout -b feature/your-feature-name
   ```

4. **Make Changes**: Implement your changes in the codebase. Ensure that you follow the coding standards outlined in this document.

5. **Write Tests**: If applicable, write tests for your changes under `src/test/oracle/pkg_uuids/` to ensure that your code works as expected.

6. **Run Tests**: Before submitting your changes, run the existing tests to ensure that everything is functioning correctly.

7. **Commit Your Changes**: Commit your changes with a clear and descriptive commit message.
   ```
   git commit -m "Add a brief description of your changes"
   ```

8. **Push to Your Fork**: Push your changes to your forked repository.
   ```
   git push origin feature/your-feature-name
   ```

9. **Submit a Pull Request**: Go to the original repository and submit a pull request. Provide a clear description of your changes and reference any relevant issues.

## Coding Standards

- Follow the existing code style and conventions used in the project.
- Use 2-space indentation in all PL/SQL files.
- Use meaningful variable and function names.
- Write clear and concise comments in English.
- Source files belong under `src/main/oracle/packages/`.
- Test files belong under `src/test/oracle/pkg_uuids/`.

## Reporting Issues

If you encounter any bugs or have feature requests, please open an issue in the GitHub repository. Provide as much detail as possible, including steps to reproduce the issue.
